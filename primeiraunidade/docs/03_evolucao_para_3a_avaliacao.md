# Evolucao para a 3a avaliacao

Depois desta entrega inicial, a terceira avaliacao deve mostrar evolucao real:

- limpar e revisar requisitos;
- detalhar tipos e datatypes;
- revisar BDD e IBD;
- adicionar visao comportamental;
- criar activities/actions;
- adicionar constraints/equations;
- adicionar executables;
- explicar estilo arquitetural;
- analisar requisitos nao funcionais.
