# O que mudou nesta versao

Nesta terceira unidade, o projeto foi organizado como evolucao direta da primeira unidade:

- a estrutura foi preservada para manter continuidade entre as entregas;
- a revisao estrutural aparece como limpeza dos requisitos, portas e responsabilidades;
- o comportamento foi colocado dentro de `SysADL_components`;
- as representacoes estruturais foram mantidas no arquivo `representations.aird`;
- foi acrescentado o estilo arquitetural Feedback Control Loop para explicar a organizacao do controle de acesso.
